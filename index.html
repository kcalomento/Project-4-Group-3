Using Machine Learning to Build a Movie Recommendation App

In today's digital era, users are overwhelmed by an ever-growing library of movies across streaming platforms. A smart recommendation system can enhance user experience by providing personalized movie suggestions based on individual preferences.

This project leverages Machine Learning and Collaborative Filtering to build a real-time movie recommendation app.

  • ✅ Data Extraction & Storage: MovieLens datasets stored in PostgreSQL (structured)
  • ✅ ML Model: Matrix Factorization and Deep Learning-based Collaborative Filtering
  • ✅ App Deployment: A Flask API serving recommendations with Redis caching

App Deployment

The app is deployed using Flask. It handles incoming user queries and interacts with the model and database, using Redis to reduce load times and PostgreSQL to manage movie data.

SQL Performance Optimization

We implemented advanced SQL caching and indexing strategies to significantly improve query performance:

Materialized Views
  • Popular Movies View: Caches top-rated movies Reduced from 3s to 0.47s
  • Recent Ratings View: Caches ratings from last 5 years Reduced from 2.8s to 0.42s
Optimization Techniques
  • Materialized View Caching
  • Targeted Indexing
  • Time-based Data Filtering
-- Sample SQL Optimization Technique
CREATE MATERIALIZED VIEW popular_movies AS
SELECT m.movieId, m.title, AVG(r.rating) AS avg_rating
FROM movies m
JOIN ratings r ON m.movieId = r.movieId
GROUP BY m.movieId
ORDER BY avg_rating DESC
LIMIT 10;
